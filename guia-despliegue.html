Vitara en línea.

Paso 2

Crear tu cuenta
GitHub

1
Abre GitHub.com
Ve a github.com en tu navegador y haz clic en "Sign up" (arriba a la derecha).
github.com
🐙 Pantalla de inicio de GitHub → busca el botón verde "Sign up"
2
Llena el formulario
Escribe tu email, crea una contraseña, y elige un nombre de usuario (por ejemplo: vitara-app). Verifica tu email cuando llegue el correo.
3
Crea un repositorio nuevo
Una vez dentro, haz clic en el botón verde "New" o "Create repository". Llénalo así:
CONFIGURACIÓN DEL REPOSITORIO
Nombre: vitara Descripción: Mi app de wellness con IA Visibilidad: Public (o Private si prefieres) README: ✓ Marcar esta casilla
Haz clic en "Create repository".
4
Sube los archivos de Vitara
En tu nuevo repositorio, haz clic en "uploading an existing file" (link azul en la página). Arrastra TODOS los archivos de Vitara que descargaste:
ARCHIVOS A SUBIR
vitara-global.html ← Landing page principal wellness-landing.html ← Página de ventas success.html ← Página post-pago onboarding.html ← Onboarding dashboard.html ← Dashboard usuario server.js ← Backend (servidor) .env.example ← Plantilla de configuración
Abajo escribe como mensaje: "Subida inicial de Vitara" y haz clic en "Commit changes".
✅
¡Listo! Tu código ya está guardado en GitHub. Ahora necesita que funcione en internet.
Paso 3

Crear tu cuenta
Stripe

💡
Importante: Stripe necesita información de tu negocio para pagarte. Ten a mano tu documento de identidad y datos bancarios.
1
Regístrate en Stripe
Ve a stripe.com → clic en "Start now". Llena con tu email y contraseña.
2
Activa tu cuenta
Stripe te pedirá información de tu negocio. Puedes empezar en modo de prueba sin llenar esto — perfecto para probar antes de cobrar en serio.
🔵
En modo de prueba puedes simular pagos con tarjetas de prueba. No se cobra dinero real hasta que actives el modo en vivo.
3
Crea tus productos en Stripe
En el panel de Stripe: ve a Products → Add product. Crea estos dos:
PRODUCTO 1 — PRO MENSUAL
Nombre: Vitara Pro Precio: $9.00 Tipo: Recurring (mensual) Guardar → copiar el Price ID: price_xxxxxxxxxxxx
PRODUCTO 2 — ANUAL
Nombre: Vitara Annual Precio: $72.00 Tipo: Recurring (anual) Guardar → copiar el Price ID: price_xxxxxxxxxxxx
4
Copia tus API Keys
Ve a Developers → API keys. Copia y guarda estos dos en un bloc de notas:
TUS CLAVES DE STRIPE (guardarlas con cuidado)
Publishable key: pk_test_... ← Esta es pública Secret key: sk_test_... ← Esta es PRIVADA, nunca la compartas
🔴
Nunca compartas tu Secret Key. Es como la contraseña de tu cuenta bancaria. Solo va en el servidor, nunca en el código del frontend.
Paso 5

Backend en
Railway

Railway va a ejecutar tu server.js — el servidor que habla con Stripe para procesar pagos.

1
Crea cuenta en Railway
Ve a railway.app → clic en "Login" → selecciona "Login with GitHub". Así Railway se conecta automáticamente a tu código.
2
Crea un nuevo proyecto
En el dashboard de Railway: "New Project" → "Deploy from GitHub repo". Selecciona el repositorio vitara que creaste.
3
Agrega un archivo package.json
Railway necesita saber cómo iniciar el servidor. Vuelve a GitHub, crea un archivo nuevo llamado package.json con este contenido exacto:
package.json
{ "name": "vitara-backend", "version": "1.0.0", "main": "server.js", "scripts": { "start": "node server.js" }, "dependencies": { "express": "^4.18.2", "stripe": "^14.0.0", "cors": "^2.8.5", "dotenv": "^16.0.0" } }
Guarda el archivo en GitHub con el mensaje "Agrega package.json".
4
Railway desplegará automáticamente
Railway detecta el cambio y construye tu servidor solo. En 2-3 minutos verás una pantalla verde con ✅. En la pestaña Settings → Domains haz clic en "Generate Domain" para obtener tu URL del servidor.
TU URL DEL SERVIDOR (ejemplo)
https://vitara-production.up.railway.app ↑ Guarda esta URL, la necesitas en el siguiente paso
Paso 5B

Variables de
entorno

Las variables de entorno son como secretos privados que le das a tu servidor sin escribirlos en el código.

1
En Railway, ve a Variables
En tu proyecto de Railway → clic en tu servicio → pestaña "Variables". Agrega cada una de estas:
Variable Valor
STRIPE_SECRET_KEY
Tu sk_test_... de Stripe
 REQUERIDA
STRIPE_PRICE_PRO
Tu price_... del plan Pro
 REQUERIDA
STRIPE_PRICE_ANNUAL
Tu price_... del plan Anual
 REQUERIDA
YOUR_DOMAIN
URL de Vercel, ej: https://vitara.vercel.app
 REQUERIDA
STRIPE_WEBHOOK_SECRET
Lo obtienes en el paso 7
 DESPUÉS
✅
Después de agregar las variables, Railway reinicia el servidor automáticamente.
Paso 6

Frontend en
Vercel

Vercel publica tus páginas HTML para que cualquier persona en el mundo pueda verlas.

1
Crea cuenta en Vercel
Ve a vercel.com → "Sign Up" → selecciona "Continue with GitHub". Acepta los permisos.
2
Importa tu repositorio
En Vercel: "Add New → Project" → busca y selecciona tu repositorio vitara de GitHub. Haz clic en "Import".
3
Configura el despliegue
Vercel detectará que es un sitio estático. Deja todo como está y haz clic en "Deploy". En 60 segundos tendrás tu URL:
TU URL DE VERCEL (ejemplo)
https://vitara.vercel.app ↑ ¡Esta es tu página web pública! Compártela con el mundo.
4
(Opcional) Agrega tu dominio propio
¿Quieres www.mivitara.com en vez de vitara.vercel.app? En Vercel → Settings → Domains → escribe tu dominio. Compra un dominio en namecheap.com por ~$10/año.
Paso 7

Activar pagos
con Stripe

1
Configura el Webhook
En Stripe: Developers → Webhooks → Add endpoint. Escribe la URL de tu servidor de Railway más /webhook:
URL DEL WEBHOOK
https://vitara-production.up.railway.app/webhook
En "Select events", elige estos eventos:
EVENTOS A ACTIVAR
checkout.session.completed customer.subscription.deleted customer.subscription.trial_will_end invoice.payment_failed invoice.payment_succeeded
Stripe te dará un Signing Secret (whsec_...). Cópialo y agrégalo a Railway como la variable STRIPE_WEBHOOK_SECRET.
Paso 8

Conectar el frontend
con el backend

Necesitas actualizar una línea en tus archivos HTML para que apunten a tu servidor de Railway.

1
Actualiza la URL del servidor
En GitHub, edita vitara-global.html, success.html y dashboard.html. Busca la línea que dice:
LÍNEA A CAMBIAR (en cada archivo HTML)
// Cambiar esto: const SERVER = "http://localhost:4242"; // Por esto (con tu URL real de Railway): const SERVER = "https://vitara-production.up.railway.app";
Guarda cada archivo con el mensaje "Conecta con servidor de producción". Vercel se actualiza automáticamente.
Paso 9

Pruebas
finales

Antes de cobrar dinero real, prueba que todo funciona con tarjetas de prueba de Stripe.

TARJETAS DE PRUEBA STRIPE
Pago exitoso: 4242 4242 4242 4242 Pago rechazado: 4000 0000 0000 9995 Requiere 3D: 4000 0025 0000 3155 Fecha: cualquier fecha futura (ej: 12/29) CVC: cualquier 3 dígitos (ej: 123)

Lista de verificación final

  • La landing page abre en mi URL de Vercel
  • El botón de "Start Free Trial" abre el checkout de Stripe
  • Puedo completar un pago con la tarjeta 4242...
  • La página success.html aparece después del pago
  • El onboarding funciona y guarda mis respuestas
  • El dashboard carga con mi nombre y plan
  • El chat con el AI Coach responde
  • En Stripe Dashboard veo el pago de prueba
Paso 10

¡Activa el modo
en vivo!

🚀
Cambia Stripe a modo Live
En Stripe, activa el toggle de Test → Live (arriba izquierda). Te pedirá completar la información de tu negocio para recibir pagos reales.
🔑
Actualiza las claves en Railway
En modo Live, Stripe te da nuevas claves que empiezan con sk_live_ y pk_live_. Actualiza las variables de entorno en Railway con estas claves reales.
⚠️
¡Cuidado! En modo Live se cobra dinero real. Asegúrate de que todo funciona en modo Test primero.
